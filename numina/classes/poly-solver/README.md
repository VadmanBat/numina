# PolySolver

**PolySolver** — современный C++ класс для нахождения **всех корней** (вещественных и комплексных) многочлена
произвольной степени с одновременным определением их **кратностей**.

Класс реализует метод Лагерра с кубической сходимостью, теорему Лагуанеля для определения кратности и последующую
полировку методом Ньютона. Поддерживает два режима дефляции: явную (по умолчанию) и неявную (процедура Мэли).

## Ключевые особенности

- **высокая точность** — достигает машинной точности даже при кратных корнях;
- **два режима работы**:
    - `Method::Explicit` — максимальная скорость (рекомендуется по умолчанию);
    - `Method::Implicit` — максимальная численная стабильность при высокой степени;
- автоматическое определение кратностей;
- корректная работа с комплексными корнями (автоматическое добавление сопряжённых пар);
- современный C++ (C++17, `noexcept`, move-семантика)

## Быстрый старт

```cpp
#include "numina/poly-solver.h"
#include <vector>
#include <iostream>

int main() {
    numina::PolySolver solver;

    // Полином (x-1)(x-2)(x-3) = x³ - 6x² + 11x - 6
    std::vector<double> coeffs = {1.0, -6.0, 11.0, -6.0};

    // Вариант 1: просто получить все корни
    auto roots = solver.solve(coeffs);

    // Вариант 2: получить корни с информацией о кратностях
    auto multi = solver.multisolve(coeffs);

    std::cout << "Корни:\n";
    for (const auto& r : roots) {
        std::cout << "  " << r << "\n";
    }

    std::cout << "\nС кратностями:\n";
    for (const auto& [val, mult] : multi.first) {
        std::cout << "  " << val << " (кратность " << mult << ")\n";
    }
    for (const auto& [val, mult] : multi.second) {
        std::cout << "  " << val << " (кратность " << mult << ")\n";
    }
}
```

## Выбор метода дефляции

| Метод                     | Когда использовать                      | Скорость | Стабильность | Рекомендация              |
|---------------------------|-----------------------------------------|----------|--------------|---------------------------|
| `Explicit` (по умолчанию) | Большинство практических задач          | ★★★★★    | ★★★★☆        | **Рекомендуется**         |
| `Implicit`                | Высокая степень, плохая обусловленность | ★★★☆☆    | ★★★★★        | При проблемах с точностью |

```cpp
// Явная дефляция (быстрее, по умолчанию)
auto roots1 = solver.solve(coeffs);

// Неявная дефляция Мэли (максимальная стабильность)
auto roots2 = solver.solve(coeffs, numina::PolySolver::Method::Implicit);
```

### Подробное сравнение режимов дефляции

| Параметр                     | Явная дефляция (`Method::Explicit`) | Неявная дефляция (`Method::Implicit`) |
|------------------------------|-------------------------------------|---------------------------------------|
| Изменение многочлена         | да (деление коэффициентов)          | нет                                   |
| Накопление ошибок округления | среднее                             | минимальное                           |
| Стабильность при μ > 1       | средняя                             | высокая                               |
| **Скорость**                 | **быстрее более чем в 3 раза**      | средняя                               |
| Асимптотика                  | $0.5 \cdot O(n^2)$                  | $O(n^2) + 0.5 \cdot O(n^2)$           |

> **Примечание**: по умолчанию используется `Method::Explicit`. Оба метода дают одинаковый результат с точностью до
> ошибок округления.

## API

Класс предоставляет два основных метода:

| Метод             | Возвращает                                                                     | Описание                                  |
|-------------------|--------------------------------------------------------------------------------|-------------------------------------------|
| `solve(...)`      | `std::vector<std::complex<double>>`                                            | Все корни (без группировки по кратностям) |
| `multisolve(...)` | `MultiRoots` (пара векторов действительных и комплексных корней с кратностями) | Корни + информация о кратностях           |

Оба метода имеют 4 перегрузки:

- `const std::vector<double>&`;
- `std::vector<double>&&`;
- `const Polynomial&`;
- `Polynomial&&`

## Известные особенности и ограничения

- определение кратности по теореме Лагуанеля работает **очень хорошо**, если корень уже уточнён до 1%;
- метод Лагерра использует фиксированное начальное приближение `0`;
- комплексные корни возвращаются в паре с сопряжёнными;
- ограничение по точности определяется типом данных расчёта. Использование `double` позволяет комфортно работать с
  полиномами **до 10-й степени и немного больше**

## Производительность и тестирование

**Общая оценка: ★★★★☆ (4.2 / 5)**

Решатель отлично справляется с:

- обычными полиномами;
- комплексными корнями;
- кратными корнями (чистыми);
- близкими корнями

**Сильные стороны:**

- почти машинная точность на случайных, комплексных, малых коэффициентах;
- идеально на чистых кратных корнях;
- надёжная обработка без NaN/Inf

**Слабые стороны (требует доработки):**

- полином Уилкинсона (ошибка до 7e-6);
- экстремальный разброс коэффициентов;
- большие по модулю корни

**Рекомендации:**

- можно использовать для большинства реальных задач;
- Для плохо обусловленных полиномов рекомендуется улучшить масштабирование или добавить fallback на companion-матрицу

Подробный отчёт: [docs/test-report.md](docs/test-report.md)
