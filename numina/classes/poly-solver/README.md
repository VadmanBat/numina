# PolySolver — математическая реализация

Класс `PolySolver` предназначен для нахождения **всех корней** (вещественных и комплексных) многочлена \( p(z) \)
произвольной степени с одновременным определением их **кратностей**.

## 1. Математическая основа

### 1.1 Метод Лагерра (Laguerre’s method)

Основной итерационный метод — **метод Лагерра (Эдмон Лагерр / Edmond Laguerre, 1880)**.  
Для многочлена степени \( n \) в точке приближения \( z \) вычисляются:

\[
G(z) = \frac{p'(z)}{p(z)}, \qquad
H(z) = G(z)^2 - \frac{p''(z)}{p(z)}
\]

Затем шаг вычисляется по формуле:

\[
L = \frac{n}{G \pm \sqrt{(n-1)(n H - G^2)}}
\]

Знак в знаменателе выбирается так, чтобы модуль знаменателя был **максимальным**.  
Метод обладает **кубической сходимостью** для простых корней и отличной глобальной сходимостью.

### 1.2 Определение кратности

Определение кратности основано на **предложении Жана-Луи Лагуанеля (Jean-Louis Lagouanelle, 1966)**.  
Теоретически кратность \( \mu \) корня \( r \) определяется по формуле:

\[
\mu = \lim_{z \to r} \frac{(z - r) \cdot p'(z)}{p(z)}.
\]

При численном вычислении на приближённом \( x \approx r \) возникает неопределённость \( 0/0 \).  
Класс разрешает её применением **правила Лопиталя (Гийом де Лопиталь / Guillaume de l'Hôpital)** последовательно.  
В реализации используются предвычисленные полиномы и вычисляется отношение

\[
\frac{[p'(x)]^2}{[p'(x)]^2 - p(x) p''(x)} \approx \mu
\]

(формула Лагуанеля). При необходимости берутся высшие производные.

### 1.3 Уточнение и полировка корня

После грубого приближения методом Лагерра выполняется **уточнение корня** с учётом уже **определённой кратности** \(
\mu \).  
**Полировка** (refinement) осуществляется **методом Ньютона (Исаак Ньютон / Isaac Newton)** на **производной полинома
порядка \( (\mu-1) \)**, где кратность искомого корня становится равной 1. Это позволяет достичь машинной точности даже
при кратных корнях.

### 1.4 Неявная дефляция (Maehly’s procedure)

После нахождения корня \( r_k \) с кратностью \( \mu_k \) многочлен **не делится** (явная дефляция).  
Вместо этого в каждой следующей итерации метода Лагерра корректируются величины \( G \) и \( H \):

\[
G_{\text{mod}} = G - \sum_{k} \frac{\mu_k}{z - r_k}, \qquad
H_{\text{mod}} = H - \sum_{k} \frac{\mu_k}{(z - r_k)^2}.
\]

В формуле Лагерра вместо полной степени \( n \) используется **эффективная степень**:

\[
n_{\text{eff}} = n - \sum_{k} \mu_k.
\]

Это математически эквивалентно применению метода Лагерра к многочлену

\[
q(z) = \frac{p(z)}{\prod (z - r_k)^{\mu_k}}
\]

без фактического деления коэффициентов. Неявная дефляция (Maehly’s procedure) обеспечивает высокую устойчивость при
больших степенях.

## 2. Структура вычислений в классе

- `f(x)`, `d1(x)`, `d2(x)` — Horner-схема на оригинальном многочлене;
- `dfx[]` — вектор производных (лениво расширяется);
- Неявная дефляция — в `solveWithImplicitDeflation()` (рекомендуется).

## 3. Сравнение двух режимов

| Параметр                     | Явная дефляция (`solveWithMultiplicities`) | Неявная дефляция (`solveWithImplicitDeflation`) |
|------------------------------|--------------------------------------------|-------------------------------------------------|
| Изменение многочлена         | да (деление)                               | нет                                             |
| Накопление ошибок округления | высокое при степени > 30–50                | минимальное                                     |
| Стабильность при μ > 1       | средняя                                    | высокая                                         |
| Скорость при высокой степени | падает из-за O(n²)                         | лучше (O(n) на итерацию)                        |

## 4. Известные особенности и ограничения

- Определение кратности основано на численном Лопитале и работает **очень хорошо**, если корень уже уточнён до ~
  10⁻¹²–10⁻¹⁴. В редких случаях (≈ 0,01 %) может потребоваться больше итераций Ньютона.
- Метод Лагерра использует фиксированное начальное приближение `(-1, -1)`.
- Комплексные корни возвращаются в паре с сопряжёнными (если коэффициенты вещественные).

## 5. Рекомендации по использованию

Для большинства задач **используйте**:

```cpp
auto roots = solver.solveWithImplicitDeflation(coefficients);
```
