# Класс Polynomial

`Polynomial` — это **высокопроизводительный** класс библиотеки **Numina** для работы с полиномиальными функциями на C++.

## Возможности

- **Представление полинома**: коэффициенты хранятся в `std::vector<double>` (от старшей степени к младшей). Автоматическое удаление ведущих нулей.
- **Типы данных**:
    - Основной тип — `Type = double`
    - Комплексные числа — `std::complex<Type>`
- **Арифметика**:
    - `+`, `-`, `*`, `/`, `%` (деление с остатком)
    - Операции с константами (скалярами)
    - Комбинированные операторы `+=`, `-=`, `*=`, `/=`, `%=`
    - Унарный `+` и `-`
- **Дифференцирование и интегрирование**:
    - `derivative()` / `makeDerivative()` (в том числе k-й порядок)
    - `integral()` / `makeIntegral()`
- **Работа с корнями**:
    - `computeRoots()` — нахождение всех корней (вещественных и комплексных)
    - `computeRootsWithMultiplicity()` — корни с кратностью
    - `deflate()` / `makeDeflate()` — деление на линейный множитель `(x - r)`
    - `deflateConjRoot()` / `makeDeflateConjRoot()` — для комплексно-сопряжённых корней
    - `monic()` / `makeMonic()` — приведение к мономическому виду
- **Дополнительно**:
    - `compose()` — композиция полиномов `p(q(x))`
    - `gcd()` — наибольший общий делитель двух полиномов (алгоритм Евклида)
    - Вычисление значения полинома в точке: `p(x)` (для `double`, `long double`, `complex`)
    - Операторы ввода/вывода: `<<` и `>>`

## Рекомендации по использованию

Класс **Polynomial** предназначен в первую очередь для работы с **исходными коэффициентами**.

- После вычисления производных, интегралов и особенно после серии арифметических операций **накапливаются ошибки округления**. Рекомендуется проводить корректировку полинома после таких операций.
- **Поиск корней** реализован в отдельном специализированном классе `PolySolver`. Методы `computeRoots()` и `computeRootsWithMultiplicity()`.
- Точность решения **существенно снижается** при наличии **кратных корней** и при **высокой степени** полинома.

**Совет**: старайтесь как можно дольше работать с исходным полиномом и минимизировать количество промежуточных производных и преобразований перед финальным вычислением корней или других критически важных результатов.
